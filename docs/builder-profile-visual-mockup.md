## Color Coding System

### Score Ranges
- **70-100**: 🟢 Green (Excellent)
- **50-69**: 🟡 Yellow (Good)
- **30-49**: 🟠 Orange (Needs Improvement)
- **0-29**: 🔴 Red (Critical)

### Assessment Types
- **PROJECT**: 🔵 Blue badge
- **PROBLEM_SOLUTION**: 🟣 Purple badge
- **VIDEO**: 🟢 Green badge
- **QUIZ**: 🟡 Yellow badge

### Status Labels
- **Top Performer**: Default badge (primary color)
- **Above Average**: Secondary badge (gray)
- **On Track**: Secondary badge (gray)
- **Needs Support**: Destructive badge (red)

## Key UI/UX Principles

1. **Progressive Disclosure**: Overview → Timeline → Details
2. **Data Hierarchy**: Big score up top, details below
3. **Visual Clarity**: Color coding, badges, clear labels
4. **Context**: Always show cohort comparison
5. **Actionable**: Links to GitHub, videos, etc.
6. **Responsive**: Works on mobile, tablet, desktop
7. **Accessible**: Screen reader friendly, keyboard navigation
8. **Performance**: Lazy load charts, cache data
